Notion API + Telegram Bot 入門 - 30分で始める業務自動化

2026.05.08 · by hanezu · 読了時間 約12分

東京藝術大学デザイン科の hanezu です。

「業務自動化を始めたいけど、どこから手をつければ？」

こういう質問が一番多い。だから今回は「30分で動くBot」を実際に作る手順を、コード付きで完全公開します。

作るもの

毎朝7時に Notion のタスクDB から今日のタスクを取得 → Telegram に通知する Bot。

これは私が3年前から運用している実物です。

必要なもの（全部無料）

  • ✅ Notion アカウント
  • ✅ Telegram アカウント
  • ✅ Node.js v18+
  • ✅ テキストエディタ（VS Code 推奨）
  • ✅ Render.com アカウント（デプロイ用・無料枠）

クレジットカード不要。30分で完成。

Step 1: Notion のセットアップ（5分）

1-1. タスクDBを用意

Notion で新しいデータベース（テーブル）を作成。プロパティは以下の3つ:

  • タスク名（タイトル）
  • 期限（日付）
  • ステータス（ステータス: Todo / Doing / Done）

1-2. Integration を作成

  1. notion.so/my-integrations にアクセス
  2. 「+ 新しいインテグレーション」をクリック
  3. 名前（例: "Daily Bot"）を入力 → 作成
  4. 「シークレット」をコピー（後で使う）

1-3. DBに Integration を接続

タスクDBのページで右上「···」→「接続を追加」→ 作成した Integration を選択。

1-4. データベースID取得

DBのURLを開く:

https://notion.so/workspace/00000000000000000000000000000000?v=...

太字部分（32桁）が Database ID。コピーする。

Step 2: Telegram のセットアップ（5分）

2-1. Bot を作成

  1. Telegram で @BotFather を検索
  2. チャットを開く → /newbot 送信
  3. Bot の名前（例: My Daily Bot）を入力
  4. Bot の username（例: hanezu_daily_bot）を入力（_botで終わる）
  5. Bot Token が発行される → コピー

2-2. 自分の Chat ID 取得

  1. 作った Bot に自分から1度メッセージを送る（"hello" 等）
  2. ブラウザで以下を開く: 
    https://api.telegram.org/botYOUR_BOT_TOKEN/getUpdates
  3. JSON の中の "chat":{"id":XXXXX} の数字をコピー

Step 3: コードを書く（10分）

3-1. プロジェクト作成

mkdir my-daily-bot
cd my-daily-bot
npm init -y
npm install @notionhq/client node-fetch@2 dotenv

3-2. .env ファイル作成

NOTION_TOKEN=secret_xxxxxxxxxxxxxxxx
NOTION_DATABASE_ID=00000000000000000000000000000000
TELEGRAM_BOT_TOKEN=0000000000:AAAAAAAAAAAAAAAAAAAAAAAAA
TELEGRAM_CHAT_ID=000000000

3-3. index.js 作成

require('dotenv').config();
const { Client } = require('@notionhq/client');
const fetch = require('node-fetch');

const notion = new Client({ auth: process.env.NOTION_TOKEN });

async function fetchTasks() {
  const today = new Date().toISOString().split('T')[0];
  const res = await notion.databases.query({
    database_id: process.env.NOTION_DATABASE_ID,
    filter: {
      and: [
        { property: '期限', date: { on_or_before: today } },
        { property: 'ステータス', status: { does_not_equal: 'Done' } }
      ]
    }
  });
  return res.results.map(p => ({
    title: p.properties['タスク名'].title[0]?.plain_text || '無題',
    url: p.url
  }));
}

async function sendTelegram(text) {
  const url = `https://api.telegram.org/bot${process.env.TELEGRAM_BOT_TOKEN}/sendMessage`;
  await fetch(url, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      chat_id: process.env.TELEGRAM_CHAT_ID,
      text,
      parse_mode: 'Markdown'
    })
  });
}

async function main() {
  const tasks = await fetchTasks();
  if (tasks.length === 0) {
    await sendTelegram('☀️ 今日のタスクはありません！');
    return;
  }
  const lines = tasks.map((t, i) => `${i + 1}. [${t.title}](${t.url})`);
  await sendTelegram(`☀️ *今日のタスク*\n\n${lines.join('\n')}`);
}

main();

3-4. ローカルで動作確認

node index.js

Telegram に通知が届いたら成功！

Step 4: 自動実行（10分）

Render.com の Cron Job 機能を使えば、無料で毎朝7時に自動実行できる。

4-1. GitHub にプッシュ

.gitignore に .env と node_modules を必ず追加してから:

git init
echo ".env\nnode_modules" > .gitignore
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/USERNAME/my-daily-bot.git
git push -u origin main

4-2. Render で Cron Job 作成

  1. Render.com でアカウント作成
  2. 「New」→「Cron Job」
  3. GitHub リポジトリを連携
  4. 設定:
    • Name: my-daily-bot
    • Build Command: npm install
    • Command: node index.js
    • Schedule: 0 22 * * * （UTC 22時 = JST 7時）
  5. 環境変数（.env の中身）を Environment Variables に転記
  6. 「Create Cron Job」

これで完成。明日の朝7時から、毎日タスクが Telegram に届く。

応用：色々な Bot に展開可能

このパターンが理解できれば、以下も全部同じ構造で作れる:

  • 📧 Gmail の重要メールを LINE に転送
  • 📊 スプレッドシート集計を Slack に日次レポート
  • 📰 RSS のニュースを Gemini で要約 → Telegram
  • 📅 Google Calendar の今日の予定を朝7時に通知
  • 🐦 Twitter のキーワード監視 → 即時通知
  • 💰 為替レートを毎時通知
  • 🎵 Spotify の新着リリース通知

すべて「データを取る → 整形する → 通知する」の3ステップ。

ハマりやすいポイント

❌ Notion API でデータが取れない

原因: Integration を DB に「接続」していない（Step 1-3）。

確認: DB ページの「···」→「接続」に Integration があるか。

❌ Telegram Bot が反応しない

原因: 自分から Bot に1度メッセージを送ってない。

確認: getUpdates で自分の chat_id が返ってくるか。

❌ 環境変数が読み込めない

原因: .env ファイルが正しい場所にない、または require('dotenv').config() を呼んでない。

❌ Render で動かない

原因: ローカルの .env は本番に反映されない。Render の Environment 設定に転記する必要がある。
